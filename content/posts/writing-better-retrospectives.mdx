회고는 감상을 적는 글이라고 생각하기 쉽지만, 저는 점점 **다음 행동을 정하는 문서**에 가깝다고 느끼게 됐습니다.

## 회고에 꼭 남기려는 항목

- 이번 주에 배운 것
- 예상과 달랐던 점
- 시간을 많이 쓴 이유
- 다음 주에 바꿔볼 한 가지

## 효과가 있었던 방식

회고를 길게 쓰기보다, 실행 가능한 문장으로 끝내는 편이 훨씬 도움이 됐습니다.

예를 들면 이렇게 적습니다.

- 다음 주에는 API 명세를 먼저 읽고 UI 작업을 시작한다.
- PR 설명에 변경 이유를 반드시 한 줄 이상 적는다.
- 막히는 시간을 40분 이상 끌지 않고 공유한다.

짧아도 반복되는 회고는 결국 나만의 작업 기준을 만들어줬습니다.
